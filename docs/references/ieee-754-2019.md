`round-float` deliberately exposes a sliver of this standard: the directed
rounding attributes (clause 4.3) as the `_down`/`_up` method pairs, correctly
rounded `sqrt` (clause 5.4.1) where the host provides it, and the `pown`
integer-power point semantics (clause 9.2: `pown(x, 0) == 1` for every `x`, the
signed zero and infinity results by parity) as the `RoundPown` kernel's special
table. The crate inherits no API shape from the standard, only the obligation
that each bound land on the correct side. Proxy literature:
[goldberg-1991](goldberg-1991.md) for the conceptual ground and
[muller-handbook-fp](muller-handbook-fp.md) for the modern treatment including
directed rounding implementation techniques.
